## 2024-05-24 - WPF Accessibility with AutomationProperties
**Learning:** In WPF, complex custom buttons (like those wrapping `ui:Card` or multiple `TextBlock` elements) cause screen readers to announce the entire visual tree or unhelpful text. `AutomationProperties.Name` and `AutomationProperties.HelpText` are crucial for providing a clean, single-intent description of interactive elements. Additionally, form controls like `ComboBox` need `AutomationProperties.LabeledBy` (or `AutomationProperties.Name`) to be properly associated with their labels since they don't use `<label for="...">` like HTML.
**Action:** Always add `AutomationProperties.Name` to buttons that contain icons or complex layouts, and ensure input controls have `AutomationProperties.Name` or `LabeledBy` set to the corresponding label element.
